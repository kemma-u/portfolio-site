InkTank

InkTank was a single person project completed as part of CareerFoundry’s UX Immersion course. The intent of the project and curriculum were to fully immerse the student in the UX Design process. This included learning all the skills, tools and processes of a UX Designer and to apply them while completing a design project from start to finish within the context of building a responsive app. InkTank is a mobile application used to help people in their tattoo decision process. The application is designed for people who are interested in getting tattoos or people who already have tattoos. Some features the application includes are browsing images and artists, locating nearby tattoo shops as well as searching for tattoo designs. The process included creating a competitive analysis, interviewing potential users to gain insight, creating user personas, then creating low- to mid- to high-fidelity wireframes to get to the final prototype.

My Process

Research

One of the first steps in the design process was creating a Competitive Analysis. The two competitors we used to compare were INKHUNTER and Tattoodo. The Competitive Analysis provided an informative insight about the already existing competitors on the market, as well served as the basis for setting up user stories.

Next was creating User Stories to help define the needs and goals of potential users of the InkTank app. This included setting up scenarios for the login/sign up process, the onboarding process, and any features I wanted to include in the InkTank app.

An additional part of the research process was interviewing potential users. The objective of this was to gain insights into the wants of users to better understand how people decide on moving forward in the tattoo decision process and where they find their inspiration for their next tattoo.

Based on the research obtained from the Competitive Analysis and User Interviews, I was then able to create User Personas to protray my target users for my product. With these User Personas, I was also able to create User Flows to help myself imagine a user using my app. Here, I was able to see an end-to-end process flow.
